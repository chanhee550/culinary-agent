## 진행 단계

- **수정 파일:** `db/database.py`, `db/models.py`, `db/repository.py`
- **내용:**
  - `ingredients` 테이블에 `expiry_date` 컬럼 추가

  - `shopping_list` 테이블 신규 (장보기 목록)
  - 전체 CRUD 함수 구현

### 2단계: 사용자 프로필 페이지 ✅ 완료
- **신규 파일:** `pages/4_profile.py`
- **내용:**
  - 숙련도 선택 (초보/중급/고급)
  - 선호 요리 종류 체크박스 (한식, 양식, 중식 등 9종)
  - 맛 선호도 체크박스 (매운맛, 단맛 등 6종)
  - 알레르기 재료 선택 (10종 기본 + 커스텀 입력)
  - 프로필 저장 및 현재 설정 요약 표시

### 3단계: 재료 관리 유통기한 추가 ✅ 완료
- **수정 파일:** `pages/2_ingredients.py`
- **내용:**
  - 재료 추가 시 유통기한 날짜 선택 (date_input)
  - 유통기한 임박 경고 배너 (3일 이내)
  - 재료 목록에 유통기한 표시 (만료: 빨강, 임박: 노랑)
  - 수정 시 유통기한 변경 가능

### 4단계: 레시피 추천 필터 적용 ✅ 완료
- **수정 파일:** `services/recipe.py`
- **내용:**

  - 알레르기 재료 절대 제외 프롬프트 적용
  - 유통기한 임박 재료 우선 활용 프롬프트 적용
  - 선호도 필터 임시 변경 지원 (cuisine_filter, taste_filter 파라미터)

### 5단계: 레시피 저장 + 단계별 가이드 + API 실패 UX ✅ 완료

- **내용:**

  - JSON + Markdown 리포트 자동 생성
  - 위반 사항별 심각도, 설명, 영향 요소 상세 분석

**공통 위반 3가지:**
1. `aria-allowed-attr` (critical) — Streamlit 사이드바의 ARIA 속성 문제 (Streamlit 프레임워크 내부)
2. `color-contrast` (serious) — 명도 대비 부족 → **CSS 수정 완료** (#888 → #595959, #666 → #555)
3. `region` (moderate) — landmark 미포함 콘텐츠 (Streamlit 프레임워크 구조적 한계)
